# Linden Heights Co-op Communication Plan

## Purpose

This document establishes how the board communicates with shareholders and residents, ensuring timely, consistent, and accessible information across all building operations.

---

## Communication Channels

### Primary Channels

**Building Status Page**
- Real-time status of building systems and assets
- Incident history and updates
- Scheduled maintenance calendar
- Accessible via web, with text/SMS integration for queries and updates

**Email (LindenHeightsBoard@GoogleGroups.com)**
- Official board communications, meeting notices, financial updates
- Monthly or quarterly newsletter summarizing building operations
- Used for anything requiring a paper trail or formal record

**WhatsApp Group**
- Quick, informal updates and reminders
- Community discussion and neighbor coordination
- Not for official notices (those go to email)

**Physical Postings**
- Lobby bulletin board for official notices
- Elevator notices for urgent or time-sensitive items
- Laundry room postings for relevant operational info
- QR codes linking to status page for digital access

**Calendar Feed (iCal/ICS)**
- Subscribable calendar for scheduled maintenance, board meetings, and social events
- Works with Google Calendar, Apple Calendar, Outlook, and most calendar apps
- Automatically updates when events are added or changed
- Available via QR code in lobby or link on status page

**Notification Preferences (Opt-in)**
- Residents can subscribe to text or email reminders for specific event types
- Managed via status page or simple signup form
- Options include: maintenance alerts, meeting reminders, social events, all updates

## Message Types and Cadences

### Urgent Communications (Immediate)
- Safety issues (gas leak, fire alarm, security concern)
- Utility outages (water, heat, electric, elevator)
- Building access issues (front door, intercom)

*Response time: Immediate posting to status page and WhatsApp, email within 1 hour, physical posting as situation allows.*

### Operational Communications (Within 24-48 hours)
- Scheduled maintenance and vendor visits
- Temporary service interruptions
- Package room or laundry room issues
- Pest control schedules

*Posted to status page when scheduled, WhatsApp reminder day-of, physical posting 48 hours in advance when possible.*

### Administrative Communications (Weekly/Monthly)
- Board meeting announcements (minimum 7 days notice)
- Meeting minutes and decisions
- Rule reminders and policy updates
- Building improvement updates

*Email with WhatsApp summary for high-interest items.*

### Financial Communications (Quarterly/Annual)
- Maintenance fee updates
- Assessment notices
- Budget summaries
- Reserve fund status

*Email only, with physical posting in lobby. Minimum 30 days notice for any fee changes per co-op bylaws.*

### Community Communications (As needed)
- Welcome notices for new shareholders
- Social events and gatherings
- Lost and found
- Recommendations (contractors, services)

*WhatsApp primary, email for formal welcomes.*

## Status Page Architecture

### Monitored Systems (Assets)
- Elevator
- Boiler/Heat
- Hot Water
- Laundry Room (washers, dryers)
- Intercom/Entry System
- Front Gate/Fire Escape
- Roof (repairs, access)

### Status Levels
- **Operational**: System functioning normally
- **Degraded**: Partially functional or reduced capacity
- **Outage**: Not functional
- **Maintenance**: Planned downtime or service

### Incident Workflow
1. Issue reported (by resident, super, or board member)
2. Status updated to reflect current state
3. Investigation/diagnosis
4. Vendor scheduled (if needed)
5. Work in progress
6. Resolution and verification
7. Post-incident summary (for significant issues)

### Integration Points
- **Text/SMS**: Query status ("text ELEVATOR to get current status"), receive subscribed alerts, board members can post updates
- **WhatsApp**: Group chat for community discussion, manual status updates for major incidents
- **Calendar feed (ICS)**: Subscribe to scheduled maintenance, board meetings, social events, inspections
- **Email**: Subscribed alerts and reminders, newsletter, formal notices
- **QR Codes**: Lobby postings link to status page, calendar subscribe, notification signup

## Resident Expectations

### What Residents Can Expect
- Acknowledgment of reported issues within 24 hours
- Status updates on ongoing incidents at least daily
- Advance notice for scheduled maintenance (48+ hours when possible)
- Clear indication of who to contact for different issue types

### How Residents Can Participate
- Report issues via status page, text, email, or super directly
- Subscribe to calendar feed for scheduled events
- Check status page before contacting board for system issues
- Use WhatsApp for community discussion, email for formal requests
